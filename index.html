Stack (LIFO)

Last-In-First-Out structure: last element inserted is first removed.

Operation Time Condition
Push O(1) Overflow: top == MAX-1
Pop O(1) Underflow: top == -1
Peek O(1) View top

Applications:

  • Function calls and recursion
  • Expression evaluation (infix to postfix)
  • Bracket matching
  • Undo/Redo operations
  • Browser back button

Queue (FIFO)

First-In-First-Out structure: first element is first removed.

Operation Time Description
Enqueue O(1) Insert at rear
Dequeue O(1) Remove from front
Peek O(1) View front/rear

Queue Types:

  • Linear Queue: Simple FIFO
  • Circular Queue: Wraps around, eliminates false overflow
  • Double Ended Queue (Deque): Insert/delete from both ends
  • Priority Queue: Elements by priority, not arrival order

Applications:

  • CPU process scheduling
  • Printer job queue
  • BFS graph traversal
  • Call center systems
  • I/O buffering

UNIT 2: Linked Lists

9 Hours

Dynamic data structures using node pointers: singly, doubly, and circular linked lists.

Singly Linked List

Chain of nodes where each node has data and pointer to next node.

Operation Time Description
Traversal O(n) Visit all nodes
Search O(n) Linear search
Insert at start O(1) New head
Insert at position O(n) After traverse
Delete O(n) Find and remove

Advantages:

  • Dynamic size allocation
  • No memory waste
  • O(1) insert/delete if position known
  • Flexible memory usage

Programs:

Linked List Operations

Doubly Linked List

Each node has prev and next pointers for bidirectional traversal.

Advantages over Singly:

  • Forward and backward traversal
  • O(1) deletion with node pointer
  • Better for undo/redo

Circular Linked Lists

Last node points back to first node, creating a ring.

Circular Singly LL:

  • No NULL pointers
  • Any node accessible from start
  • Round-robin scheduling

Circular Doubly LL:

  • Bidirectional with circularity
  • Most flexible traversal

UNIT 3: Tree Structures

8 Hours

Binary Search Trees, AVL Trees, B+ Trees, and traversal methods.

Binary Search Tree (BST)

Binary tree: left child < parent < right child. Enables efficient sorted data operations.

Operation Avg Time Worst Time
Search O(log n) O(n)
Insert O(log n) O(n)
Delete O(log n) O(n)
Traversal O(n) O(n)

Tree Traversal Methods

Method Order Use Case
In-Order Left - Root - Right Sorted output
Pre-Order Root - Left - Right Tree copy
Post-Order Left - Right - Root Tree deletion
Level-Order (BFS) Level by level Serialization

AVL Tree (Self-Balancing BST)

Automatically maintains balance. Height difference between subtrees <= 1.

  • Balance Factor: height(left) - height(right) in {-1, 0, 1}
  • Rotations: LL, RR, LR, RL
  • Guaranteed O(log n) operations

B+ Tree

Multi-way tree optimized for disk I/O. All data in leaves, internal nodes for routing.

  • Used in databases (MySQL, SQLite)
  • Efficient range queries
  • Minimizes disk seeks

UNIT 4: Graph Structures

13 Hours

Graph representations, traversals, shortest paths algorithms.

Graph Representations

Property Adjacency Matrix Adjacency List
Space O(V²) O(V+E)
Add Edge O(1) O(1)
Check Edge O(1) O(degree)
Best For Dense graphs Sparse graphs

Graph Traversals

Breadth-First Search (BFS):

  • Level-by-level using queue
  • Time: O(V+E)
  • Shortest path (unweighted), connectivity check

Depth-First Search (DFS):

  • Deep search using stack/recursion
  • Time: O(V+E)
  • Cycle detection, topological sort, connected components

Shortest Path Algorithms

Dijkstra (Single-Source):

  • Time: O((V+E)logV) with min-heap
  • No negative weights
  • GPS navigation, routing protocols

Floyd-Warshall (All-Pairs):

  • Time: O(V³), Space: O(V²)
  • Works with negative weights (no cycles)
  • All-pairs shortest paths
Algorithm Time Negative Weights
Dijkstra O((V+E)logV) No
Floyd-Warshall O(V³) Yes
Bellman-Ford O(VE) Yes

UNIT 5: Searching & Sorting

12 Hours

Efficient searching and sorting algorithms with complexity analysis.

Searching Algorithms

Algorithm Time Space Sorted Required
Linear Search O(n) O(1) No
Binary Search O(log n) O(1) Yes
Hashing O(1)* O(n) No

Linear Search: O(n)

  • Works on unsorted data
  • Simple but slow for large arrays

Binary Search: O(log n)

  • Divide search space in half
  • Requires sorted array
  • Fast for large datasets

Hashing: O(1)*

  • Hash function maps keys to memory
  • Collision handling: chaining or open addressing
  • Frequent lookups

Sorting Algorithms

Algorithm Avg Worst Space Stable
Selection O(n²) O(n²) O(1) No
Bubble O(n²) O(n²) O(1) Yes
Insertion O(n²) O(n²) O(1) Yes
Merge O(nlogn) O(nlogn) O(n) Yes
Quick O(nlogn) O(n²) O(logn) No
Heap O(nlogn) O(nlogn) O(1) No

When to Use:

Quick Sort: Default choice, fast in practice
Merge Sort: Need guaranteed O(nlogn) or stability
Heap Sort: In-place O(nlogn), extra space limited
Insertion/Bubble: Nearly sorted data OR small arrays only

Real-World Usage:

  • Java: Arrays.sort() uses Timsort (hybrid)
  • Python: sorted() uses Timsort
  • C++: std::sort() uses introsort (hybrid)
  • Databases: Merge sort for external sorting, B+ trees for indexing
