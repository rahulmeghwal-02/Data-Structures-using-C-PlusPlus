Arrays

An array is a linear, homogeneous data structure that stores a fixed-size sequential collection of elements of the same data type in contiguous memory locations. It is the simplest and most widely used data structure.

O(1)
Random Access
O(n)
Search (unsorted)
O(n)
Insert / Delete

Memory Representation

Array elements are stored in contiguous memory. For a 1D array A[n] with base address B and each element taking W bytes:

Address of A[i] = B + i × W

For a 2D array A[r][c] stored in row-major order (C++ default):

Address of A[i][j] = B + (i × c + j) × W

In column-major order (used in Fortran):

Address of A[i][j] = B + (j × r + i) × W

Operations on Arrays

Operation Time Description

// C++ Array Operations
int A[6] = {10, 20, 30, 40, 50, 60};
int n = 6;

// 1. Traversal
for (int i = 0; i < n; i++) cout << A[i] << " ";

// 2. Insertion at position pos
void insert(int A[], int& n, int pos, int val) {
    for (int i = n; i > pos; i--) A[i] = A[i-1]; // shift right
    A[pos] = val; n++;
}

// 3. Deletion at position pos
void deleteAt(int A[], int& n, int pos) {
    for (int i = pos; i < n-1; i++) A[i] = A[i+1]; // shift left
    n--;
}

Advantages & Limitations

Pros

• O(1) random access by index
• Simple memory layout
• Cache-friendly (spatial locality)
• Best for fixed-size data

Cons

• Fixed size (static arrays)
• O(n) insert/delete
• Memory waste if underused
• Homogeneous only

Records (Structures in C++)

A record (called struct in C++) is a composite data structure that groups variables of different data types under a single name. Each variable inside is called a field or member.

📌 Unlike arrays (homogeneous), a record/struct can hold mixed types: int, float, char, string all together.

Declaration & Memory Layout

Memory for a struct is allocated based on its members plus possible padding for alignment. 

struct Student {
    int    roll;      // 4 bytes
    char   name[30]; // 30 bytes
    float  marks;    // 4 bytes
    char   grade;    // 1 byte + 3 padding
}; // Total: ~42+ bytes (with alignment)

// Creating objects
Student s1;                          // single record
Student s1 = {101, "Rahul", 95.5, 'A'}; // initialise
Student arr[50];                      // array of records

// Accessing members — dot operator
s1.roll  = 102;
s1.marks = 88.5;
cout << s1.name << " : " << s1.grade;

Stack

A stack is a linear data structure that follows the LIFO (Last In, First Out) principle — the element inserted last is the first to be removed. Think of a stack of plates: you always add and remove from the top.

LIFO
Principle
O(1)
Push & Pop
TOP
Active end

Operations on Stack

Operation Description Condition Check
push(x) Insert element x at top Fails if FULL (overflow)
pop() Remove top element Fails if EMPTY (underflow)
peek() / top() View top without removing Fails if EMPTY
isEmpty() Check if stack is empty Returns true if TOP == -1
isFull() Check if stack is full Returns true if TOP == MAX-1
size() Number of elements Returns TOP + 1

Stack Conditions

// Stack Overflow  → push on a full stack
if (top == MAX - 1) { cout << "Overflow!"; return; }

// Stack Underflow → pop/peek on empty stack
if (top == -1)     { cout << "Underflow!"; return; }

Applications of Stack

Function Call Stack

Each function call pushes a frame; return pops it. Enables recursion.

Expression Evaluation

Infix → Postfix conversion; evaluating postfix expressions.

Undo / Redo

Text editors push states. Undo = pop. Two stacks for redo support.

Bracket Matching

Compilers check ( ) [ ] { } balance using a stack.

Backtracking

Maze solving, DFS graph traversal, game state management.

Browser History

Back button = pop current page and return to previous.

Stack Implementation using Array

An array-based stack uses a fixed-size array and a top variable (initialised to -1) to track the topmost element. It is the simplest and most efficient implementation.

📌 top = -1 means the stack is empty. top = MAX-1 means the stack is full.

Complete C++ Implementation

#include <iostream>
using namespace std;

#define MAX 100   // maximum stack size

class Stack {
private:
    int arr[MAX];
    int top;

public:
    Stack() { top = -1; }  // constructor

    // Check if empty
    bool isEmpty() {
        return top == -1;
    }

    // Check if full
    bool isFull() {
        return top == MAX - 1;
    }

    // Push operation — O(1)
    void push(int x) {
        if (isFull()) {
            cout << "Stack Overflow!\n";
            return;
        }
        arr[++top] = x;   // increment top, then store
        cout << x << " pushed\n";
    }

    // Pop operation — O(1)
    int pop() {
        if (isEmpty()) {
            cout << "Stack Underflow!\n";
            return -1;
        }
        return arr[top--];  // return top, then decrement
    }

    // Peek / Top — O(1)
    int peek() {
        if (isEmpty()) {
            cout << "Stack is empty!\n";
            return -1;
        }
        return arr[top];
    }

    // Size
    int size() { return top + 1; }

    // Display
    void display() {
        cout << "Stack (top→bottom): ";
        for (int i = top; i >= 0; i--)
            cout << arr[i] << " ";
        cout << endl;
    }
};

int main() {
    Stack s;
    s.push(10);  s.push(20);  s.push(30);
    s.display();            // 30 20 10
    cout << s.peek();      // 30
    cout << s.pop();       // 30 removed
    s.display();            // 20 10
    return 0;
}

Infix → Postfix using Stack

A classic application of stack in expression conversion:

// Convert A+B*C to ABC*+
// Rules: operand → output; operator → compare precedence with stack top
// Precedence: * / > + -   ( has lowest when in stack

int prec(char op) {
    if (op == '*' || op == '/') return 2;
    if (op == '+' || op == '-') return 1;
    return 0;
}
// Example: A+B*C → scan left to right
// A  → output: A
// +  → push:   [+]          output: A
// B  → output: AB
// *  → * > +, push: [+,*]  output: AB
// C  → output: ABC
// end→ pop all: ABC*+

Queue

A queue is a linear data structure that follows the FIFO (First In, First Out) principle — the element inserted first is the first to be removed. Think of a real-life queue at a ticket counter.

FIFO
Principle
REAR
Insert end
FRONT
Delete end

Operations on Queue

Operation Description Affected Pointer
enqueue(x) Insert x at REAR REAR = REAR + 1
dequeue() Remove from FRONT FRONT = FRONT + 1
front() View FRONT element No change
rear() View REAR element No change
isEmpty() Check empty FRONT > REAR
isFull() Check full (array) REAR == MAX-1

Types of Queues

Simple Linear Queue

FRONT and REAR move in one direction. Once REAR reaches end, no more insert even if space at front (false overflow). Array-based.

Circular Circular Queue

REAR wraps around to index 0 after reaching MAX-1. Eliminates false overflow. Best array implementation.

Double Deque (Double Ended)

Insert and delete from BOTH ends. Two variants: input-restricted and output-restricted deque.

Priority Priority Queue

Elements dequeued based on priority, not arrival order. Used in scheduling, Dijkstra's algorithm.

Queue Implementation using Array

Uses a fixed-size array with two pointers: front (initialised to -1) and rear (initialised to -1). Insert at rear, delete from front.

Linear Queue — C++ Code

#define MAX 100

class Queue {
private:
    int arr[MAX], front, rear;

public:
    Queue() { front = rear = -1; }

    bool isEmpty() { return front == -1; }
    bool isFull()  { return rear == MAX - 1; }

    // Enqueue — insert at REAR, O(1)
    void enqueue(int x) {
        if (isFull()) {
            cout << "Queue Overflow!\n"; return;
        }
        if (isEmpty()) front = 0;  // first element
        arr[++rear] = x;
        cout << x << " enqueued\n";
    }

    // Dequeue — remove from FRONT, O(1)
    int dequeue() {
        if (isEmpty()) {
            cout << "Queue Underflow!\n"; return -1;
        }
        int val = arr[front];
        if (front == rear) front = rear = -1; // last element
        else front++;
        return val;
    }

    int  getFront() { return arr[front]; }
    int  getRear()  { return arr[rear];  }
    int  size()     { return rear - front + 1; }
};

Circular Queue — Solving False Overflow

In a linear queue, after many enqueue/dequeue operations, REAR reaches MAX-1 even though front positions are empty. Circular queue reuses those positions.

📌 REAR = (REAR + 1) % MAX    FRONT = (FRONT + 1) % MAX 
class CircularQueue {
private:
    int arr[MAX], front, rear, count;

public:
    CircularQueue() { front = 0; rear = -1; count = 0; }

    bool isEmpty() { return count == 0; }
    bool isFull()  { return count == MAX; }

    void enqueue(int x) {
        if (isFull()) { cout << "Full!\n"; return; }
        rear = (rear + 1) % MAX;  // wrap around!
        arr[rear] = x;
        count++;
    }

    int dequeue() {
        if (isEmpty()) { cout << "Empty!\n"; return -1; }
        int val = arr[front];
        front = (front + 1) % MAX;  // wrap around!
        count--;
        return val;
    }
};

State Trace — Queue Operations

Operation Array front rear Notes
Init [ ] -1 -1 Empty
enqueue(10) [10] 0 0 front set to 0
enqueue(20) [10,20] 0 1 rear++
enqueue(30) [10,20,30] 0 2 rear++
dequeue() [_,20,30] 1 2 Returns 10, front++
dequeue() [_,_,30] 2 2 Returns 20, front++

Applications of Queue

Queues are fundamental to systems where order of arrival matters — fairness, scheduling, and sequential processing.

🖨️ Print Spooler / Job Scheduling

Print jobs are added to a queue. The printer processes them in FIFO order — first submitted, first printed. OS job schedulers (Round Robin) use circular queues for CPU time slots.

🌐 BFS (Breadth-First Search)

Graph/tree traversal uses a queue to explore nodes level by level. Start node enqueued → dequeue → enqueue all unvisited neighbours → repeat. Used in shortest path algorithms.

📞 Call Center / Helpdesk

Incoming calls are queued when agents are busy. First caller waits less. Queue ensures no one is skipped unfairly — fairness through FIFO.

💾 IO Buffers / Keyboard Buffer

Characters typed are stored in a keyboard buffer queue. They are processed by the OS in the order typed, ensuring correct character sequence display.

🔄 Inter-Process Communication

Message queues (like POSIX MQ, RabbitMQ) allow processes/threads to communicate. Producer enqueues messages; consumer dequeues. Used in OS pipelines.

🎮 Simulation Systems

Bank simulations, airport runway scheduling, event-driven systems — all model real-world queues to study wait times and service efficiency.
